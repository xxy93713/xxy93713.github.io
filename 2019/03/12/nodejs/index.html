一.Node.js是什么？

-    简单的说 Node.js 就是运行在服务端的 JavaScript。Node.js基于Chrome v8引擎，相当于在node里写js和在谷歌浏览器里写js一样。
-    Node.js 是一个基于Chrome JavaScript 运行时建立的一个平台。
-    Node.js是一个事件驱动I/O服务端JavaScript环境，基于Google的V8引擎，V8引擎执行Javascript的速度非常快，性能非常好。
-    Node.js可以做什么（如：文件的读写，进程的管理，网络通信），这都是Chrome浏览器做不了的。

三．为了让Node.js的文件可以相互调用，Node.js提供了一个基于CommonJS的模块系统(commonJS主要应用于服务器端编程)，这里我们简单讲一下CommonJS规范：
1、CommonJs规范的出发点：由于JS没有模块系统、标准库较少、缺乏包管理工具；为了让JS可以在任何地方运行，以达到Java、PHP这些后台语言具备开发大型应用的能力；
2、在CommonJs规范中：
· 一个文件就是一个模块，拥有单独的作用域；
· 普通方式定义的变量、函数、对象都属于该模块内, 其属于私有的，对其他文件不可见；
· 通过exports和modul.exports来暴露模块中的内容；（Node开发者建议导出对象用module.exports,导出多个方法和变量用exports）
· 通过require来加载模块；（Node的模块可分为两类：Node提供的核心模块；用户编写的文件模块）
3、所有代码都运行在模块作用域，不会污染全局作用域；模块可以多次加载，但只会在第一次加载的时候运行一次，然后运行结果就被缓存了，以后再加载，就直接读取缓存结果；模块的加载顺序，根据代码的出现顺序同步加载的;（Node核心模块加载速度仅次于缓存中加载，然后路径形式的模块次之，最慢的是自定义模块）
4、dirname代表当前模块文件所在的文件夹路径，filename代表当前模块文件所在的文件夹路径+文件名;
5、require（同步加载）基本功能：读取并执行一个JS文件，然后返回该模块的exports对象，如果没有发现指定模块会报错;
6、模块内的exports：为了方便，node为每个模块提供一个exports变量，其指向module.exports，相当于在模块头部加了这句话：var exports = module.exports，在对外输出时，可以给exports对象添加方法，PS：不能直接赋值（因为这样就切断了exports和module.exports的联系）;
四. NPM是随同NodeJS一起安装的包管理器，能解决NodeJS代码部署上的很多问题，常见的使用场景有：
（1）允许用户从NPM服务器下载别人编写的第三方包到本地使用。
（2）允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
（3）允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

  1. NPM 使用介绍：由于新版的nodejs已经集成了npm，所以之前npm也一并安装好了，若是不确定Npm是否安装成功，同样可以通过输入 “npm -v” 来测试是否成功安装；
    若之前安装的是旧版本的npm，直接npm install npm –g 升级即可。
    1）npm root –g => 查看npm全局包安装位置，建议在nvm目录下新建npm\node_modules目录，然后设置npm的全局包安装位置：npm config set prefix “”，然后将该路径添加到环境变量中;
    2）npm init –y => 初始化一个package.json文件，加上-y就会默认生成该文件，无需一步一步填写；
    3）npm docs 包名：查看包的文档；
    4）安装 Node.js 的模块 : npm install -g // -g是全局安装，不加-g是本地安装 => npm install 可在本地安装当前所有命令行程序
    5）若出现以下错误：npm err! Error: connect ECONNREFUSED 127.0.0.1:8087 => 解决办法：npm config set proxy null
    6）查看所有全局安装的模块 : npm list –g
    7）查看某个模块的版本号 : npm list
    8）卸载某个Node.js 模块：npm uninstall
    9）查看包是否还存在:npm ls
    10）更新某个或全部模块：npm updata （-g）
    11） 搜索某个模块：npm search
    12） 在npm 资源库里注册用户：npm adduser
    13） 注册完发布模块：npm publish => 成功注册后，可以像其他模块一样使用npm 来安装
    14）查看npm 的所有命令：npm help = > npm help install 查看install命令的详细帮助
    15）清空npm 本地缓存：npm cache clear => 用于对付使用相同版本号发布新版本代码的人；
    16）npm unpublish @ => 用来撤销发布自己发布过的某个版本代码。
    17）.NPM的模块加载机制：安装好之后，安装包放在 ./node_modules 下，如果没有 node_modules 目录，会在当前执行 npm 命令的目录下生成 node_modules 目录。可以通过 require() 来引入本地安装的包，如果require的是绝对路径文件，查找不会去遍历每个node_modules目录，其速度最快
    2.由于npm的服务器是国外的，所以如果没有和谐工具是下载不了的：（替代方法）
    1）这里推荐使用淘宝NPM镜像：http://npm.taobao.org/，与官方NPM的同步频率目前为10分钟一次；
    安装命令：npm install -g cnpm –registry=https://registry.npm.taobao.org，安装包：cnpm install 包名（其它命令基本一致）;
    2）如果你不想下载cnpm，npm还提供了一个镜像源管理工具：npm install -g nrm，通过：nrm ls，查看镜像源列表 ，通过：npm use 镜像源，来切换;
    五．Packeage.json属性说明：
    1）name - 包名。
    2）version - 包的版本号。
    3）description - 包的描述。
    4）homepage - 包的官网 url 。
    5）author - 包的作者姓名。
    6）contributors - 包的其他贡献者姓名。
    7）dependencies - 依赖包列表。如果依赖包没有安装，npm 会自动将依赖包安装在 node_module 目录下。
    8）repository - 包代码存放的地方的类型，可以是 git 或 svn，git 可在 Github 上。
    9）main - main 字段指定了程序的主入口文件，require(‘moduleName’) 就会加载这个文件。这个字段的默认值是模块根目录下面的 index.js。
    10）keywords - 关键字
    六．创建一个服务器：
    1．在创建 Node.js 第一个服务器之前，我们先了解下 Node.js 应用的几个组成部分：
    （1）引入 required 模块：我们可以使用 require 指令来载入 Node.js 模块。
    （2）创建服务器：服务器可以监听客户端的请求。
    （3）接收请求与响应请求 ：服务器创建后，客户端可以使用浏览器或终端发送 HTTP 请求，服务器接收请求后返回响应数据。
    先了解下http模块：大体来说Http模块，主要应用是两部分，一部分是http.createServer 担当web服务器，另一部分是http.createClient，担当客户端。
    http服务器：实例如下，在你本地根目录下创建一个index.js 的文件，并写入以下代码：
    const http = require(‘http’);
    const hostname = ‘127.0.0.1’;
    const port = 8888;
    const server = http.createServer((req, res) => {
      res.statusCode = 200; // HTTP 状态值: 200 : OK
  res.setHeader('Content-Type', 'text/plain'); // 内容类型: text/plain
//res.writeHead(200,{‘content-Type’,’text/plain’})
  res.end('Hello World');
    });
    server.listen(port, hostname, () => {
    console.log(`服务器运行在 http://${hostname}:${port}/`);
    });
    // server.close([callback]) // 停止服务端接收新的连接
    //Node.jsHttp模块也提供了server.timeout用于查看或设置超时
    // server.timeout = 1000 //设置超时为1秒

使用 node 命令执行以上的代码：node index.js

  1. request对象
    • request.url 客户端请求的url地址
    • request.headers 客户端请求的http header
    • request.method 获取请求的方式，一般有几个选项，POST,GET和DELETE等，服务器可以根据客户端的不同请求方法进行不同的处理。
    • request.httpVersion http的版本
    • request.trailers 存放附加的一些http头信息
    • request.socket 用于监听客户端请求的socket对象
    我们可以写一小段js讲客户端的请求信息保存在log.txt中（例子：req.js）
    可以看到我们引入了一个fs模块（fs 模块提供了一些接口用于与文件系统进行交互，所有的文件系统操作都有同步和异步两种形式，用法和api官网有写）
  2. response对象
    • response.writeHead(statusCode, [reasonPhrase], [headers])
    • response.statusCode html页面状态值
    • response.header 返回的http header，可以是字符串，也可以是对象
    • response.setTimeout(msecs, callback)设置http超时返回的时间，一旦超过了设定时间，连接就会被丢弃
    • response.statusCode 设置返回的网页状态码
    • response.setHeader(name, value)设置http协议头
    • response.headersSent判断是否设置了http的头
    • response.write(chunk, [encoding]) 返回的网页数据，[encoding] 默认是 utf-8
    • response.end([data], [encoding])
    4.URL解析
    在Node.js中，提供了一个url模块与querystring (查询字符串)模块
    （1）querystring模块 用于 URL 处理与解析
    querystring.parse(str, [sep], [eq], [options]) // 将字符串转成对象
    querystring.parse(‘foo=bar&baz=qux&baz=quux&corge’)
    // returns
    { foo: ‘bar’, baz: [‘qux’, ‘quux’], corge: ‘’ }
    querystring.stringify(obj,separator,eq,options) //这个方法是将一个对象序列化成一个字符串，与querystring.parse相对
    querystring.stringify({name: ‘whitemu’, sex: [ ‘man’, ‘women’ ] });
    // returns
    ‘name=whitemu&sex=man&sex=women’
    （2）url 模块提供了一些实用函数，用于 URL 处理与解析
    一个 URL 字符串是一个结构化的字符串，它包含多个有意义的组成部分。 当被解析时，会返回一个 URL 对象，它包含每个组成部分作为属性。
    url.format(urlObject) 回一个从 urlObject 格式化后的 URL 字符串 (类似JSON.stringfy)
    url.parse( ) 解析一个 URL 字符串并返回一个 URL 对象(类似JSON.parse)_
    { protocol:’http:’, auth: ‘user:pass’, hostname:‘host.com’, post:‘8080’，pathname:‘/p/a/t/h’,search:{query=string}, hsah：‘#hash’}
    url.resolve(from,to) 基于from根路径，对最后一个字符做转换to；
    七．Path模块：
    1)path.join([…paths])方法用于连接路径 // path.join的参数必须为字符串
    2) path.resolve() 方法会把一个路径或路径片段的序列解析为一个绝对路径方法用于将相对路径转为绝对路径
    path.resolve(‘wwwroot’, ‘static_files/png/‘, ‘../gif/image.gif’)
    // 如果当前工作目录为 /home/myself/node，
    // 则返回 ‘/home/myself/node/wwwroot/static_files/gif/image.gif’
    3)path.extname() 方法返回 path 的扩展名，即从 path 的最后一部分中的最后一个 .（句号）字符到字符串结束
    path.extname(‘index.html’)
    // 返回: ‘.html’

    九．Node中的stream (流)
    1. Writable 可写流 => 例如：fs.createWriteStream
      常用事件：1）readable 在数据块可以从流中读取的时候发出
      2）data  有数据可读时发出
3）end  当数据被读完时发出，对应的处理器没有参数。
4）close 当底层的资源，如文件，已关闭时发出
5）error 当在接收数据中出现错误时发出。
      2.Readable - 可读的流 => 例如：fs.createReadStream()
      3.Duplex - 可读写的流 => 例如：net.Socket
      4.Transform - 在读写过程中可以修改和变换数据的 Duplex 流 => 例如： zlib.createDeflate()
    2. 管道流：用于从一个流中获取数据并将数据传递到另外一个流中
      6.链式流：链式是通过连接输出流到另外一个流并创建多个对个流操作链的机制
      const fs = require(“fs”);
      const zlib = require(‘zlib’)
      // 压缩 README.md 文件为 README.md.gz
      fs.createReadStream(‘./README.md’)
      .pipe(zlib.createGzip())
      .pipe(fs.createWriteStream(‘README.md.gz’))
      console.log(“文件压缩完成”)
